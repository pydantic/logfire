# Pydantic Logfire

Logfire is an OpenTelemetry-native observability platform for tracing, metrics, evaluations, dashboards, alerts, and AI application monitoring.

Use this documentation to choose the right SDK and then learn the platform features that work across languages.

## Choose an SDK

- [Python SDK](python-sdk/index.md) - instrument Python applications with manual spans, logs, metrics, auto-instrumentation, framework integrations, and the generated API reference.
- [TypeScript SDK](https://pydantic.dev/docs/logfire/typescript-sdk/) - instrument JavaScript and TypeScript applications across Node.js, browsers, Cloudflare Workers, and OpenTelemetry-compatible runtimes.
- [Other languages](languages.md) - send OpenTelemetry data from Rust, Go, Java, .NET, and any language with OTLP support.

## Learn the platform

- [Core Concepts](concepts.md) explains traces, spans, metrics, projects, and OpenTelemetry terminology.
- [Observe & Investigate](guides/web-ui/live.md) covers Live View, dashboards, SQL Explorer, issues, alerts, saved searches, and public traces.
- [Evaluate](guides/web-ui/evals.md) covers datasets, experiments, live evals, and evaluation workflows.
- [Manage & Configure](guides/web-ui/organizations-and-projects.md) covers organizations, projects, write tokens, API keys, environments, regions, and cost controls.
- [Deploy & Scale](enterprise.md) covers enterprise, compliance, audit logs, SSO, and self-hosted deployment.
